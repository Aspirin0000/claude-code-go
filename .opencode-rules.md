# OpenCode Agent 硬性规则（不可违反）

## 黄金法则（最高优先级）
1. **零简化原则**：禁止生成简化版、框架版、占位符版代码
2. **完整实现原则**：所有代码必须完整可运行，禁止 "// TODO" 和 "..."
3. **分块优于简化**：文件过大时必须分块处理，而非简化实现

## 禁止行为清单（触发即失败）
- [ ] 生成"核心接口框架"代替完整实现
- [ ] 使用"暂时简化"或"占位实现"
- [ ] 创建 Provider 接口推迟实际实现
- [ ] 删除"复杂验证逻辑"或"非核心功能"
- [ ] 用 300 行框架代替 3000 行完整代码
- [ ] 任何包含 "TODO", "FIXME", "stub", "placeholder", "simplified" 的代码

## 正确处理模式（必须遵循）
| 场景 | 错误做法 | 正确做法 |
|------|---------|---------|
| 文件太大 (1579行) | 简化实现 | 分块完整实现 (1-200, 201-400...) |
| 依赖未就绪 | 创建接口占位 | 推迟文件，先处理依赖 |
| 时间有限 | 提取"核心功能" | 完整实现当前批次，剩余批次后续处理 |
| 代码复杂 | 生成框架让用户填充 | 逐行完整迁移，保持所有逻辑 |

## 自检咒语（每个文件生成后必须执行）
生成代码后立即自检：
- 行数是否 < 原文件 50%？→ 重新实现
- 是否包含禁用词 (TODO/stub/...)? → 删除并补充
- 是否有未实现的函数体 {}? → 补充完整逻辑
- 是否删除了原代码的验证/错误处理？→ 恢复

## 违规处理流程
如果检测到违规：
1. 立即停止生成
2. 删除违规代码
3. 输出："检测到简化行为，重新完整实现..."
4. 重新开始完整实现

## 记忆强化（每次启动时复述）
启动时输出："已加载零简化规则，执行完整重构模式"
